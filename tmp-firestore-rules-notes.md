# Firestore Rules Notes

## Codebase findings

- Platform: single-page web app in `public/index.html` using Firebase Auth and Firestore modular SDK loaded from CDN.
- Auth pattern: Google sign-in with Firebase Auth. Firestore access is user-scoped.
- Existing private document path: `users/{userId}`.
- Existing Firestore read: `getDoc(doc(db, "users", user.uid))`.
- Existing Firestore write: `setDoc(doc(db, "users", user.uid), { name, email, envelope, updatedAt }, { merge: true })`.
- New snapshot collection path: `users/{userId}/monthly_snapshots/{monthId}`.
- New snapshot query: `query(collection(db, "users", user.uid, "monthly_snapshots"), orderBy("month", "desc"), limit(24))`.
- New snapshot write: `setDoc(doc(db, "users", user.uid, "monthly_snapshots", monthId), snapshot, { merge: true })`.
- Monthly close strategy: client-side rollover on the first access of a new month, with the previous month becoming immutable after sync.

## Data model assumptions

- `users/{userId}` stores private profile data plus the full finance envelope and must remain owner-only.
- `envelope.updatedAt` and `users.updatedAt` are ISO-8601 strings generated by the client.
- `envelope.data.settings` contains the structured monthly income and benefits base.
- Lists inside `envelope.data` are user-owned and can be large, so the rules prototype limits list sizes but does not deeply validate every nested item yet.
- `users/{userId}/monthly_snapshots/{monthId}` stores one document per month with:
  - `userId`
  - `month`
  - `income`
  - `totalGastos`
  - `totalDividas`
  - `comprometimento`
  - `status`
  - `source`
  - `closedAt`
  - `createdAt`
  - `updatedAt`

## Access assumptions

- No public Firestore reads.
- A user can only read and write their own user document.
- A user can only read snapshot documents inside their own `monthly_snapshots` subcollection.
- Client writes are limited to `source == "client"` snapshots inside the owner scope.
- A client may close an `open` snapshot once, but cannot modify or delete a `closed` snapshot afterward.
- Snapshot document ids are the canonical month key in `YYYY-MM`.
